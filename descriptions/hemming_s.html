Код Хеммінга. Короткі теоретичні відомості.

    Код Хеммінга - один з найбільш поширених систематичних кодів. До кодів Хеммінга відносяться коди з мінімальною кодовою відстанню dmin = 3, що виправляють всі поодинокі помилки.

    Формування r перевірочних елементів в комбінації цього коду здійснюється по kінформаційним елементам. Таким чином, довжина кодової комбінації n = r + k. Перевірочні елементи являють собою лінійні комбінації інформаційних елементів, тобто зважені суми інформаційних елементів з ваговими коефіцієнтами 1 і 0. Для двійкових кодів як лінійної операції використовують додавання по модулю 2. Операція віднімання збігається з операцією додавання.

    Послідовність 0 і 1 в кодової комбінації інакше називають кодовою вектором. Коду Хеммінга притаманні властивості лінійних кодів:

  1. Сума або різниця кодових комбінацій коду дає вектор, що належить даному коду.

  2. Лінійні коди утворюють алгебраїчну групу по відношенню до операції додавання по модулю 2.

  3. Мінімальна кодова відстань між кодовими векторами групового коду дорівнює мінімальній вазі ненульових кодових векторів.

    При передачі кодового вектора може бути спотворений будь-який елемент, число таких ситуацій дорівнює C1n = n. До цього слід додати ще одну ситуацію, коли помилка не відбулася. Таким чином, загальне число комбінацій перевірочних елементів 2r має перевищувати число можливих помилкових ситуацій в коді з урахуванням відсутності помилок для правильного їх відмінності і визначення місць помилок: 2r >= n + 1. Оскільки 2n = 2r+k = 2k2r, можна записати: 2n >= ( n + 1 ) 2k, де 2n – повне число комбінацій коду.

    Мінімальне співвідношення коригувальних і інформаційних розрядів, нижче якого код не може зберігати свої коригувальні здібності, визначається з виразу 2r - 1 = n.

    Для обчислення основних параметрів коду Хеммінга можна задати число перевірочних елементів r, коли з останнього виразу визначають n и k = n - r. Співвідношення між r, n и k для коду Хеммінга наведені в таблиці:

r23333444444445
n345678910111213141516
k11234456789101111

    Характерною особливістю перевірочної матриці коду з dmin = 3 є те, що її стовпці представляють собою різні ненульові комбінації довжиною r. Наприклад, при r = 4 и n = 15, код (15, 11), перевірочна матриця має вигляд:

H15,11 = 000000011111111
000111100001111
011001100110011
101010101010101
u1u2u3u4u5u6u7u8u9u10u11u12u13u14u15

    Отже, якщо взяти комбінації простого чотирьохелементного коду і відкинути нульову комбінацію, то можна легко отримати перевірочну матрицю, записавши всі кодові комбінації послідовно в стовпці матриці H.

    Перестановкою стовпців, що містять одну одиницю, вправо матрицю приводимо до виду:

H15,11 = 000011111111000
011100011110100
101101100110010
110110101010001
a1a2a3a4a5a6a7a8a9a10a11a12a13a14a15

    Відповідно до цієї матриці, отримуємо систему перевірочних рівнянь, за допомогою якої обчислюємо перевірочні розряди:

b1 = a5 (+) a6 (+) a7 (+) a8 (+) a9 (+) a10 (+) a11
b2 = a2 (+) a3 (+) a4 (+) a8 (+) a9 (+) a10 (+) a11
b3 = a1 (+) a3 (+) a4 (+) a6 (+) a7 (+) a10 (+) a11
b4 = a1 (+) a2 (+) a4 (+) a5 (+) a7 (+) a9  (+) a11

    При появі помилки в кодовій комбінації виявляються невиконаними ті перевірочні відносини, в які входить значення помилкового розряду.

    Так, якщо помилка виникла в 7-му інформаційному розряді, то виявляться невиконаними 1, 3 і 4 рівняння, тобто синдром дорівнює 1011 (збігається з 7-м стовпцем матриці H). Таким чином, місце розташування стовпця матриці H, що збігається з обчисленим синдромом, вказує місце помилки. Обчислення значення синдрому обов'язково збігається з одним з стовпців матриці H, тому що в якості стовпців обрані всі можливі виконавчі r-розрядні числа.

    Хеммінг запропонував розташувати стовпці перевірочної матриці так, щоб номер i-го стовпця матриці H і номер помилкового розряду кодової комбінації відповідали бінарного поданням числа i. Тоді синдром, отриманий з перевірочних рівнянь, буде двійковим поданням номера розряду кодової комбінації, в якому сталася помилка. Для цього перевірочні розряди повинні знаходитися не в кінці кодової комбінації, а на номерах позицій, які виражаються ступенем двійки ( 20, 21, 22, ... 2r-1 ), як це має місце в непреобразованной матриці H, тому що кожен з них входить тільки в одне з перевірочних рівнянь.

    В останньому випадку перевірочні розряди розміщуються між інформаційними. Синдром, згідно непреобразованной перевірочної матриці H15,11, визначається з рівнянь:

S1 =u1(+) u3 (+) u5 (+) u7 (+) u9 (+) u11 (+) u13(+) u15
S2 = u2 (+) u3 (+) u6 (+) u7 (+) u10 (+) u11 (+) u14(+) u15
S3 = u4 (+) u5 (+) u6 (+) u7 (+) u12 (+) u13 (+) u14(+) u15
S4 = u8 (+) u9 (+) u10 (+) u11 (+) u12 (+) u13  (+) u14(+) u15

    В якості перевірочних вибирають розряди u1, u2, u4 и u8, які  зустрічаються  в цій системі рівнянь по одному разу.

    Наприклад, якщо необхідно   закодувати повідомлення простого коду 11001010110 (k = 11) в коді Хеммінга, потрібно визначити перевірочні розряди в комбінації: u1 u2 1 u4 100 u8 1010110. З перевірочної матриці отримуємо:

u1 = u3 (+) u5 (+) u7 (+) u9 (+) u11 (+) u13(+) u15
u2 = u3 (+) u6 (+) u7 (+) u10 (+) u11 (+) u14(+) u15
u4 = u5 (+) u6 (+) u7 (+) u12 (+) u13 (+) u14(+) u15
u8 = u9 (+) u10 (+) u11 (+) u12 (+) u13  (+) u14(+) u15

    Обчислюючи перевірочні розряди згідно з цією системою, отримаємо для даного повідомлення:

u1 = 1 (+) 1 (+) 0 (+) 1 (+) 1 (+) 1 (+) 0 = 1,
u2 = 1 (+) 0 (+) 0 (+) 0 (+) 1 (+) 1 (+) 0 = 1,
u4 = 1 (+) 0 (+) 0 (+) 0 (+) 1 (+) 1 (+) 0 = 1,
u8 = 1 (+) 0 (+) 1 (+) 0 (+) 1 (+) 1 (+) 0 = 1.

    Отже, комбінація коду Хеммінга для розглянутого повідомлення матиме вигляд 111110001010110.

    Припустимо, що 8-й єлемент кодової комбінації прийнятий помилково, тобто отримано повідомлення 111110011010110. Обчислюючи синдром, отримаємо: S1 = 0; S2 = 0; S3 = 0; S4 = 1,  тобто синдром має вигляд 10002 = 810 (очевидно, що для отримання правильного результату біти синдрому необхідно записати, починаючи з старшого, тобто S4S3S2S1). Таким чином, необхідно виправити 8-й разряд кодової принятої комбинації.
