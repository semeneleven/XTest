Двійково-десятковий код. Короткі теоретичні Відомості.

      У двійково-десятковому коді кожна десяткова цифра представляється групою цифр, що складається з 4-х двохпозиційних символів. Дана група дозволяє сформувати N = 2 4 = 16 кодових комбінацій. Так як в десятковій системі використовується лише десять цифр, шість комбінацій є надлишковими. Тому вибір десяти використовуваних для побудови ДДК комбінацій має 16! · 6! = 2.9 · 10 10 варіантів. Використання, наприклад, перших чотирьох ступенів цифри 2 (2 0 = 1; 2 1 = 2; 2 2 = 4; 2 3 = 8) призводить до одного з можливих кодів 8-4-2-1.
      Кожен розряд ДДК має постійну вагу. ДДК будуються з урахуванням наступних умов:

      
  1. Вага найменшої значущої цифри q 1 дорівнює 1.
    2.   
  2. Вага другий за мінімальним значенням цифри q 2 становить 1 або 2.
    3.   
  3. Ваги, що відповідають двом залишилися цифрам коду, підбирають так, щоб їх сума була більше або дорівнює 6 (якщо q 2 = 2), або 7 (якщо q 2 = 1).

      У відповідності з цим можна отримати 17 видів кодів:
8-4-2-1, 7-4-2-1, 6-4-2-1, 5-4-2-1, 4-4-2-1,
7-3-2-1, 6-3-2-1, 5-3-2-1, 4-3-2-1, 3-3-2-1,
6-2-2-1, 5-2-2-1, 4-2-2-1, 6-3-1-1, 5-3-1-1,
4-3-1-1, 5-2-1-1.

      Для перекладу однієї десяткової цифри в двійковій-десятковий код необхідно починаючи з старшого довічного розряду перевірити, чи не більше вага поточної (двійковій) цифри залишку числа; якщо менше або дорівнює - то в вихідний код записати 1 і відняти вагу двійковій цифри з залишку числа, інакше в вихідний код записати 0

      Наприклад, необхідно закодувати цифру 5 кодом 6-4-2-1.

  1. Вага самого старшого біта - 6, що більше поточного залишку (початкове значення якого збігається з числом). Таким чином, перша цифра двійковій-десяткового коду   - 0.
  2. Вага 2-го біта - 4> 5. Значить, наступний біт вихідного коду дорівнює 1 і залишок дорівнює 5 - 4 = 1.
  3. Вага 3-го біта - 2, що більше, ніж залишок (1). У вихідний код поміщається 0.
  4. Вага 4-го біта - 1 дорівнює залишку числа. Остання цифра вихідного коду   - 1. Остаточний залишок 1 - 1 = 0. Його рівність 0 говорить про правильність кодування.

      Результат кодування - 0101

      Для отримання значення тетради (4 біт), записаної в двійковій-десятковому коді, необхідно підсумувати значення ваг розрядів, значення яких дорівнюють 1.

      Раскодіруем число 0101 з 6-4-2-1 довічного-десяткового коду. Результат: 0 · 6 + 1 · 4 + 0 · 2 + 1 · 1 = 5.

      ДДК не мають однозначності в зображенні десяткових чисел (крім коду 8-4-2-1). Так, код 4-3-2-1 дозволяє записати число 64 як 1010 1000 або 0111 0101.

      Приклади кодування.

      

  1. Закодувати число 8701 двійковій-десятковим кодом 5-2-1-1.

    Так як:
    8 = 5 · 1 + 2 · 1 + 1 · 1 + 1 · 0;
    7 = 5 · 1 + 2 · 1 + 1 · 0 + 1 · 0;
    0 = 5 · 0 + 2 · 0 + 1 · 0 + 1 · 0;
    1 = 5 · 0 + 2 · 0 + 1 · 1 + 1 · 0,
    закодоване число матиме вигляд:     1110 1100 0000 0010.

    2.   

  2. Закодувати число 927 двійковій-десятковим кодом 7-4-2-1.

    Так як:
    9 = 7 · 1 + 4 · 0 + 2 · 1 + 1 · 0;
    2 = 7 · 0 + 4 · 0 + 2 · 1 + 1 · 0;
    7 = 7 · 1 + 4 · 0 + 2 · 0 + 1 · 0,
    закодоване число буде мати вигляд     1010 0010 1000.

      Приклади декодування.

      

  1. Декодувати число 001001011100, якщо відомо, що воно закодовано в двійковій-десятковому коді 6-3-2-1.

    Так як:
    6 · 0 + 3 · 0 + 2 · 1 + 1 · 0 = 2;
    6 · 0 + 3 · 1 + 2 · 0 + 1 · 1 = 4;
    6 · 1 + 3 · 1 + 2 · 0 + 1 · 0 = 9,
    то береться стверджувати числом буде 249.

    2.   

  2. Декодувати число 100101110010, якщо відомо, що воно закодовано в двійковій-десятковому коді 8-4-2-1.

    Так як:
    8 · 1 + 4 · 0 + 2 · 0 + 1 · 1 = 9
    8 · 1 + 4 · 1 + 2 · 1 + 1 · 1 = 7
    8 · 0 + 4 · 0 + 2 · 1 + 1 · 0 = 2 < br> то береться стверджувати числом буде 972.
