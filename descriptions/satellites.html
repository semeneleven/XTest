Коды-спутники. Краткие теоретические сведения.

Метод исправления ошибок с помощью использования кодов-спутников.

    Систематическими называются коды, у которых проверочные элементы определяются в результате проведения линейных операций над определенными информационными элементами. Данный метод исправления ошибок в систематическом коде предусматривает построение кодовой таблицы с кодами-спутниками.В случае приема комбинации, совпадающей с одним из кодов-спутников, искаженная комбинация расшифровывается как исходная рабочая комбинация, в которую входит данный код-спутник. Тестирование проходит в два этапа. В режиме кодирования на экране появляется шестизначное число в двоичной форме. Необходимо в приведенную таблицу занести все его коды-спутники с dmin = 1 и dmin = 2. Количество этих кодов всегда равно 21. В режиме декодирования необходимо выбрать из таблицы числа с наименьшим dmin.

Построение кодовой таблицы

    Кодовая таблица с кодами-спутниками строится по следующему правилу: в первой строке располагаются все кодовые комбинации Vj, для которых необходимо найти коды-спутники; во второй строке записывают векторы, которые получают в результате суммирования по mod2 кодовых комбинаций Vj с вектором e1, вес которого w = 1, а единица расположена в первом разряде; в третьей строке – результат суммирования по mod2 Vj с вектором e2, вес которого w = 1, а единица распоположена во втором разряде и т.п. Так поступают до тех пор, пока не будут просуммированы с кодовыми комбинациями Vj все векторы ei весом w = 1 с единицами в каждом из N разрядов; затем суммируются по mod2 векторы Vj с весом w = 2 с последовательным перекрытием всех возможных разрядов (вес векторов ej определяет число исправляемых ошибок, а его разрядность соответствует разрядности кодовой комбинации). Таким образом, для каждой кодовой комбинации Vi систематического кода получают свою группу кодов-спутников, расположенных в соответствующем столбце. В случае приема комбинации, совпадающей с одним из кодов-спутников, искаженная комбинация расшифровывается как исходная рабочая комбинация, в которую входит данный код-спутник. Недостатками данного метода являются высокие требование к памяти и временные затраты на перебор комбинаций кода-спутника.
